Configuration Management System:

Configuration Management System ensures a consistent, secure and reliable way to manage the system state, due to ever growing technological complexity, configuration management tools provide a simple yet effective platform to manage these complex systems. CM increases stability, efficiency and control over the system.

Just like we test our apps, backup the state regularly, version controlled, it’s very crucial to consider the same approach with the systems, neglecting would lead to huge risks, delayed deployments, and huge costs.

Ansible:

Ansible tries to be simple and is built on existing concepts and which are widely known, hence it has relatively low learning curve compared to other popular CM tools.

JAnsible doesn’t require any software agent to be installed on target machines, only requirement is to have access to the target system, hence no need to upgrade management agents, not to worry about patching security vulnerabilities in those agents.

Ansible has feature called dry run, which basically means to preview the changes that will happen when the script gets executed, accidental modifications will be minimized.

Ansible core distribution has over 700 modules and over 4,000 community-provided roles which are easy to read and easy to run immediately.

Ansible Tower:

Ansible Tower is a hub for all of automation tasks, a visual dashboard reporting:

  • Server deployments
  • Failures
  • Health
  • History of playbooks executed
  • Different layers of security and authorization
  • User management
  • Inventory management

Tower helps to conceptualize the state of a machine in a more intuitive way.

It also can schedule jobs, notifies about state of the system. Tower provides a REST API and CLI to interact and extend as desired.
